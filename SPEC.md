# SPEC — SteadyTranscribe v1

Эталон: функция File/Meeting Transcription из FluidVoice (карта функции изучена по исходникам, поведение воспроизводится 1:1 с адаптацией под Windows). UI на русском.

## Модуль 1: Транскрипция файла

### User Stories
- Как пользователь, я хочу перетащить аудио/видео-файл в окно и получить текст, чтобы не печатать вручную
- Как пользователь, я хочу видеть прогресс с процентами и стадией, чтобы понимать, сколько ждать
- Как пользователь, я хочу, чтобы всё происходило локально, чтобы записи не покидали компьютер
- Как пользователь, я хочу скопировать или экспортировать результат
- Как пользователь, я хочу увидеть понятную ошибку, если файл не поддерживается

### Модель данных (SQLite: %APPDATA%/SteadyTranscribe/history.db)
```sql
entries: id (TEXT PK, uuid4), timestamp (TEXT NOT NULL, iso8601),
         file_name (TEXT NOT NULL), duration (REAL), processing_time (REAL),
         confidence (REAL), text (TEXT NOT NULL), model (TEXT), language (TEXT)
-- лимит 50 записей: при вставке 51-й удаляется самая старая (как maxEntries=50 в оригинале)
-- пустой text не сохраняется (как в оригинале)
```

### API (внутренние интерфейсы)
```
core.convert.to_wav16k(path) → tmp_wav_path | ConvertError      # ffmpeg: любой аудио/видео → 16kHz mono wav
core.transcriber.transcribe(wav, cb) → {text, confidence, duration, processing_time} | TranscribeError
storage.history.add(entry) / list() / delete(id) / clear()
storage.settings.get()/set(key)                                  # JSON %APPDATA%/SteadyTranscribe/settings.json
```

### Экраны и компоненты (главное окно, одна колонка карточек — как в оригинале)
- **Шапка**: иконка + «Транскрипция SteadyControl» + «Перетащите аудио- или видеофайл»
- **Карточка выбора файла**: состояния `пусто` (пунктирная зона, ярче при drag-over; клик = диалог выбора) | `выбран` (имя + размер + ✕ сброс + кнопка «Расшифровать», disabled при работе)
- **Карточка прогресса** (только при работе): полоса + статус. Стадии как в оригинале: 10% «Подготовка модели…», 20% «Анализ файла…», 30–90% «Распознавание… N%», 100% «Готово!»
- **Карточка результата**: «Расшифровка готова» + статистика (длительность c, уверенность %, скорость ×) + кнопки Копировать / Экспорт + текст (скролл, выделяемый)
- **Карточка ошибки**: красная, текст + «Скрыть»; ошибка формата — авто-скрытие 3 с
- **История** («Недавние расшифровки» + «Очистить всё»): строки имя/относительное время/превью 80 символов; выбранная — детальная карточка (дата полностью, Копировать/Экспорт/Удалить)
- **Тост** «Скопировано!» — авто-скрытие 2 с
- Состояния: loading (прогресс) | error (карточки ошибок) | empty (истории нет → секция скрыта) | success

### Бизнес-логика
- Форматы: wav, mp3, m4a, aac, ogg, opus, flac, wma, mp4, mov, mkv, avi, webm (ffmpeg сам достаёт аудиодорожку из видео)
- Пайплайн: файл → ffmpeg → wav 16kHz mono → faster-whisper (streaming по сегментам, длинные файлы без ручного chunking — но с VAD-фильтром) → текст
- confidence = средняя exp(avg_logprob) по сегментам; скорость = duration/processing_time
- Модель: tiny/base/small/medium/large-v3-turbo (по умолчанию **small**, int8); скачивается при первом использовании в %APPDATA%, с прогрессом; **зеркало HuggingFace настраивается** (авто-fallback на hf-mirror.com — сеть Олега блокируется HF)
- Настройки: модель, язык (авто/ru/en/…), устройство (auto/cpu/cuda), словарь подсказок (initial_prompt), лимит истории
- Дубликаты не отслеживаются (как в оригинале). Отмена: кнопка «Отменить» при работе (улучшение — в оригинале нет, но нужна: файлы длинные, Windows-CPU медленный)

### Крайние случаи (поведение как в оригинале)
- Неподдерживаемое расширение при drop → красная карточка «Поддерживаются: …», авто-скрытие 3 с
- ffmpeg не смог декодировать → ошибка «Не удалось обработать аудио»
- Модель не скачана и нет сети → понятная ошибка + подсказка про зеркало в настройках
- Пустой результат (тишина) → в историю не пишется, показывается «Речь не обнаружена»
- Файл 3+ часа → работает (потоковая обработка), прогресс по сегментам
- Закрытие окна во время работы → подтверждение

## Экспорт
- **TXT**: шапка `Transcription: <имя>` / `Date:` / `Duration:` / `Processing Time:` / `Confidence:` / `---` / текст (формат оригинала)
- **JSON**: prettyPrinted+sortedKeys: text, confidence, duration, processingTime, fileName, timestamp (iso8601)
- Имя по умолчанию: `<имя>_transcript.txt|.json`, системный диалог сохранения

## Упаковка
- PyInstaller (onedir) + ffmpeg.exe в комплекте → Inno Setup: один установщик `SteadyTranscribe-Setup.exe`, ставит в Program Files, ярлыки, деинсталлятор. Модели НЕ в установщике (скачиваются при первом запуске)
- Сборка: GitHub Actions windows-latest, артефакт — установщик
