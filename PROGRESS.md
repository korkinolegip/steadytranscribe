## Как выпускать релиз (чек-лист)
1. Правки → py_compile + pyflakes (гейт в CI дублирует)
2. ИЗМЕНЕНИЯ UI → прогнать workflow ui-screens, СМОТРЕТЬ снимки глазами (правило Олега)
3. Поднять версию в ТРЁХ местах: updater.py CURRENT_VERSION, installer.iss AppVersion, version_info.txt (гейт в CI проверит совпадение с тегом)
4. При пользе для пользователя — строка в ui/changelog.py (по-человечески)
5. commit → push main → tag vX.Y.Z → push tag; лёгкий установщик публикуется ~10 мин
6. Крупные изменения установщика → прогнать test-update.yml
